Google Cloud Platform (GCP) Quick Start

Ref: 2026 | Cloud Console & gcloud CLI

1. Cloud Console Navigation

Erreichbar unter: console.cloud.google.com

  • Projekt-Selector: Oben links neben dem GCP-Logo. Jede Ressource muss einem Projekt zugeordnet sein.
  • Navigationsmenü: Das "Burger-Menü" (≡) öffnet alle Dienste (Compute, Storage, Networking).
  • Suchleiste (/) : Schnellster Weg zu Diensten (z.B. "IAM" oder "Bucket").
AktionTastenkürzel
Suche fokussieren/
Cloud Shell öffnenG dann S
Dashboard anzeigenG dann D

2. Wichtige Konzepte

RegionGeografischer Ort (z.B. europe-west3 Frankfurt).
ZoneIsolierter Ort innerhalb einer Region (z.B. -a, -b).
IAMIdentity & Access Management (Wer darf was?).
QuotasLimits für Ressourcen pro Projekt/Region.

3. Cloud Shell & gcloud CLI

Die Cloud Shell ist ein integriertes Terminal mit vorinstalliertem SDK.

Basis-Konfiguration:
Logingcloud auth login
Projekt setzengcloud config set project [ID]
Status prüfengcloud config list
Compute Engine (VMs):
VM auflistengcloud compute instances list
SSH Zugriffgcloud compute ssh [NAME]
VM Start/Stopgcloud compute instances start [N]
Cloud Storage (gsutil):
Bucket erstellengsutil mb gs://[NAME]
Datei Uploadgsutil cp [FILE] gs://[BUCKET]
Inhalt listengsutil ls gs://[BUCKET]

4. Best Practices & Troubleshooting

Sicherheit:
  • Nutze das Principle of Least Privilege (IAM).
  • Vermeide den Owner-Status für Service Accounts.
  • Nutze Infolayouts zur Kostenüberwachung.
Hilfe:
  • Hilfe zu jedem Befehl: gcloud [COMMAND] --help
  • Aktivitäts-Log: In der Console unter "Activity" prüfen, warum Aktionen fehlgeschlagen sind.
GCP OnePager - Optimiert für den Druck. Benutze "Hintergrundgrafiken drucken" in den Browsereinstellungen für beste Ergebnisse.
