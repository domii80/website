Die Architektur moderner KIs

Ein strukturierter Überblick über die Transformer-Technologie. Die bahnbrechende Neuerung der Transformer (erstmals 2017 im Paper "Attention Is All You Need" vorgestellt) liegt im Verzicht auf rekurrente neuronale Netze (RNNs) zugunsten des reinen Einsatzes von Aufmerksamkeitsmechanismen (Attention). Dadurch wird die vollständige Parallelisierung bei der Verarbeitung von Sequenzen möglich.

1. Tokenisierung & Embeddings

Bevor Text verarbeitet werden kann, muss er in den Vektorraum abgebildet werden.

  • Tokenization: Der Text wird in diskrete Einheiten (Tokens) zerlegt (oft via Byte-Pair Encoding oder WordPiece).
  • Input Embeddings: Jedes Token wird über eine Embedding-Matrix auf einen hochdimensionalen, dichten Vektor x ∈ ℝd abgebildet. Dies erfasst die semantische Bedeutung des Tokens.

2. Positional Encoding

Da Transformer keine rekurrente Struktur haben, verarbeiten sie alle Tokens simultan (Permutationsinvarianz). Dem Modell fehlt nativ das Konzept von Wortreihenfolgen.

  • Mechanismus: Dem Input-Embedding wird ein Vektor gleicher Dimension addiert, der die Position des Tokens codiert.
  • Implementierung: Klassisch via Sinus- und Kosinus-Funktionen; modernere Modelle nutzen oft gelernte Embeddings oder RoPE (Rotary Positional Embeddings).

3. Scaled Dot-Product Attention (Self-Attention)

Das absolute Herzstück des Transformers. Es gewichtet in Echtzeit den Kontext aller anderen Tokens für das aktuell betrachtete Token.

  • Aus dem Embedding entstehen drei Vektoren: Query (Q) ("Was suche ich?"), Key (K) ("Was biete ich?") und Value (V) ("Was ist mein Inhalt?").
  • Berechnung: Die Ähnlichkeit wird über das Skalarprodukt von Q und K bestimmt und skaliert.

4. Multi-Head Attention (MHA)

Anstatt Attention nur einmal auszuführen, passiert dies parallel in mehreren "Heads".

  • Mechanismus: Matrizen werden in kleinere Unterräume projiziert und die Attention parallel ausgeführt.
  • Nutzen: Erlaubt dem Modell, gleichzeitig verschiedene Aspekte (z. B. Grammatik vs. Semantik) zu fokussieren. Ergebnisse werden am Ende konkateniert.

5. Position-wise Feed-Forward Networks (FFN)

Nach der Attention durchläuft die Repräsentation jedes Tokens unabhängig ein klassisches Multi-Layer Perceptron.

  • Aufbau: Zwei lineare Transformationen mit einer nicht-linearen Aktivierungsfunktion (heute meist GELU oder SwiGLU).
  • Zweck: Dient der Feature-Transformation und Verarbeitung der internen Repräsentationen des individuellen Tokens.

6. Residual Connections & Layer Normalization

Standardtechniken aus dem Deep Learning stabilisieren das Training in tiefen Netzen.

  • Residual Connections (Add): Die Eingabe einer Sub-Schicht wird über einen Shortcut auf deren Ausgabe addiert, um den Vanishing Gradient zu verhindern.
  • Layer Normalization (Norm): Die Werte werden anschließend über die Feature-Dimension normalisiert.
xout = LayerNorm(x + Sublayer(x))

7. Masked Attention & Auto-Regressive Generierung

Transformer teilen sich grob in zwei Hauptkategorien:

  • Encoder (z. B. BERT): Bidirektionale Attention. Jedes Token schaut "nach links" und "rechts". Ideal für Klassifizierung.
  • Decoder (z. B. GPT, Gemini, LLaMA): Textgenerierung. Das Modell darf bei der Vorhersage nicht in die "Zukunft" schauen.
  • Masking: Die obere rechte Dreiecksmatrix in der Attention-Berechnung wird mit −∞ maskiert, wodurch Informationsfluss aus zukünftigen Tokens blockiert wird.

Der Architektur-Flow eines modernen (Decoder-Only) LLMs

Zusammenfassend durchläuft ein Prompt bei der Textgenerierung folgenden Pfad:

Eingabe-Text → Tokenisierung → Embeddings + Positional Encoding →
[ N × Transformer-Blöcke: (Masked MHA → Add & Norm → FFN → Add & Norm) ] →
Lineare Projektion → Softmax (Vorhersage nächstes Token)
